# Security Policy

## Supported Versions

Security fixes are expected to land on the current `main` branch and, when practical, in the most recent App Store release line.

## Reporting a Vulnerability

Do not report security issues in public GitHub issues, discussions, or pull requests.

Use GitHub's private vulnerability reporting for this repository if it is enabled. If private reporting is not available, contact the maintainers privately before any public disclosure.

When reporting an issue, include:

- a clear description of the problem
- affected app versions or commit ranges
- reproduction steps or proof-of-concept details
- impact assessment

Maintainers will triage reports, confirm severity, and coordinate a fix and disclosure timeline.
